💡 项目介绍

本系统是一个基于大语言模型（LLM）的内容质量评估智能体，用于判断文本治理（清洗、规范化等）的质量。

从以下五个维度对比治理前后文本，给出科学评分：

  • 语义保真度（权重 0.30）—— 治理后是否保留了原意？信息是否丢失？
  • 事实一致性（权重 0.30）—— 是否发生事实篡改或关键事实丢失？
  • 幻觉检测（权重 0.20）—— 是否凭空编造了原文没有的信息？
  • 结构质量（权重 0.10）—— 表格/列表/段落等结构化信息是否被破坏？
  • 可读性（权重 0.10）—— 句子是否通顺、排版是否规范？

综合判定：

  • ≥ 0.82 → 通过 (pass)
  • 0.5~0.82 → 需复审 (review)
  • < 0.5 → 不通过 (fail)

📋 评估模式

系统支持三种评估模式，适用于不同类型的文本：

模式 适用场景 特点
通用文本 普通说明文、常规治理文本 保持现有四维度评分逻辑
政务通告/公告（严格保真） 政务通告、公告、制度通知、执法告知 重点关注日期、期限、罚则区间、适用范围等关键事实是否被删除或泛化
法规合同/条款（严格保真） 法规、合同、条款、协议、制度规范 重点关注责任主体、义务、条件、例外与法律后果是否被模糊化

严格模式额外规则：选择政务通告或法规合同严格模式后，系统会自动执行关键事实缺失检查——从原文中提取日期、金额区间、百分比、时限等关键事实，检查改写后是否仍然保留。若关键事实丢失，自动追加瑕疵并降分；多个关键事实缺失时，直接判定为 fail。

📖 操作流程

  1. 确保后端服务已启动（运行 python_version\run.bat）
  2. 在 「核心评估」 标签页输入治理前和治理后的文本
  3. 点击 「开始评估」 按钮
  4. 查看各维度评分、瑕疵清单和综合判定
  5. 如需更可靠的结果，可勾选 「启用多次采样稳定化」
