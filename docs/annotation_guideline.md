# 内容保真度与治理质量评估 —— 人工标注规范（Annotation Guideline）

**版本**：v1.1  
**日期**：2026-06-29  
**适用范围**：本规范用于"内容保真度与治理质量评估智能体"项目的人工标注、LLM Judge 评分校准、与人工一致性评价、训练集 Gold Label 生成及论文实验。

---

## 第零章：通用原则（General Principles）

在进入具体评分规则之前，所有标注员和 Judge 必须遵守以下五条通用原则。

### Principle 1：保守原则（Conservative Principle）

**不能确定有错，不给低分。**

如果标注员对某个问题是否属于错误存在犹豫，应倾向于不扣分。只有确信存在问题时才扣分。这一原则确保标注的严谨性，避免过度惩罚。

### Principle 2：最小修改原则（Minimum Modification Principle）

**只评价治理导致的问题，不评价原文本来就有的问题。**

如果治理前文本本身存在错别字、语法错误、格式不规范等问题，治理后文本不需要修正这些问题，也不应因未修正而扣分。标注对象是"治理过程引入的问题"，而非"原文固有的问题"。

### Principle 3：独立原则（Independence Principle）

**每个维度独立评分，不因其他维度的分数而调整。**

四个维度（语义一致性、过度清洗、可读性与结构、事实准确性）各自独立打分。一个维度的低分不能导致另一个维度的分数被人为降低。

### Principle 4：一致性原则（Consistency Principle）

**同一种错误，永远同一种评分。**

标注员在不同时间、不同样本上遇到相同类型的错误时，必须给出相同等级的评分。建议标注员定期回顾已标注样本，确保自身一致性。

### Principle 5：证据优先原则（Evidence-First Principle）

**所有扣分，必须找到锚点证据，否则不给扣分。**

每一条扣分都必须附带具体的文本证据（原文片段、位置信息）。无法定位到具体位置的问题，不能作为扣分依据。这一原则确保标注的可验证性和可复现性。

## 第一章：评分维度定义

本体系包含四个评分维度，各维度之间**互不重叠**，每个错误只影响其归属维度，不重复扣分。

### 维度 1：语义一致性（Semantic Consistency）

**评价对象**：治理后文本是否能够表达与治理前文本**相同的信息意图（Communicative Intent）和命题意义（Propositional Meaning）**。

**核心问题**：治理后文本是否传达了与治理前文本相同的"谁对谁做了什么、在什么条件下、产生了什么结果"。

**归属问题**：
- 核心观点/结论是否改变
- 主题是否偏移
- 因果关系、条件关系是否被扭曲
- 推理链条是否完整
- 信息意图是否改变（如从"通知"变成"建议"）

**不归属**（由其他维度覆盖）：
- 具体数值/日期/名称的准确性 → 归属"事实准确性"
- 信息是否被删除 → 归属"过度清洗"
- 句子是否通顺 → 归属"可读性与结构"

**判断方法**：对比治理前后文本的**命题意义**——即文本所断言的客观事实和逻辑关系。如果治理后文本断言了不同的事实或逻辑关系，则语义一致性低。注意：语义一致性评价的是"说了什么"，不是"说全没有"（后者归属过度清洗）。

---

### 维度 2：过度清洗（Over-Cleaning）

**评价对象**：治理过程中是否**删除了有价值的信息**，导致信息密度下降。

**归属问题**：
- 删除具体措施、执行细节
- 删除量化目标、数据指标
- 删除限定条件、前提假设
- 删除补充说明、括号注释
- 删除豁免条款、例外情况
- 合理的格式压缩（如去掉多余空格）不算过度清洗

**不归属**：
- 信息被篡改（数值错误）→ 归属"事实准确性"
- 信息被扭曲（语义改变）→ 归属"语义一致性"
- 格式/结构问题 → 归属"可读性与结构"

**判断方法**：治理后文本相比治理前，丢失了多少有价值的信息？"有价值"指对理解主题、做出决策、执行操作有帮助的信息。格式性冗余（多余空格、重复标点）的删除不算过度清洗。

---

### 维度 3：可读性与结构（Readability & Structure）

**评价对象**：治理后文本的**表达是否通顺、格式是否规范、结构是否清晰**。

**归属问题**：
- 句子是否通顺、是否有语法错误
- 段落是否连贯、逻辑是否清晰
- 表格、列表、标题等结构是否完整
- 排版是否规范
- 表格是否被转为纯文本
- 列表是否被压缩为段落
- 标题层级是否合理

**不归属**：
- 删除了什么内容 → 归属"过度清洗"
- 数值/名称是否正确 → 归属"事实准确性"
- 整体含义是否改变 → 归属"语义一致性"

**判断方法**：治理后文本的表达和格式是否便于阅读和理解？结构化数据（表格、列表）是否保留了其结构优势？

---

### 维度 4：事实准确性（Factual Accuracy）

**评价对象**：治理后文本中**仍然保留的信息**是否与治理前文本中的对应信息**事实一致**。

**关键限定**：事实准确性仅评价治理后文本中仍然保留的信息是否真实，**不评价因删除导致的信息缺失**。信息缺失归属"过度清洗"。

**归属问题**：
- 保留的数值（金额、百分比、数量）是否与原文一致
- 保留的日期（生效日期、截止日期、时间范围）是否与原文一致
- 保留的专有名词（人名、机构名、地名）是否与原文一致
- 保留的技术参数、规格指标是否与原文一致
- 保留的法律条文引用是否与原文一致

**不归属**：
- 信息被删除（没有提到）→ 归属"过度清洗"
- 整体含义是否改变 → 归属"语义一致性"
- 表达是否通顺 → 归属"可读性与结构"

**判断方法**：逐项核对治理后文本中仍然存在的具体事实，是否与治理前文本一致。任何一处保留的事实被篡改（哪怕只是小数点后一位的变化）都属于事实准确性问题。但如果某个事实被完全删除（治理后文本中没有提到），则归属过度清洗，不归属事实准确性。

## 第三章：评分等级

每个维度使用以下 5 个离散分值，不允许使用中间值。

### 1.00 —— 完全合格

**定义**：该维度无任何问题。

**判断标准**：
- 语义一致性：信息意图和命题意义完全保留，无任何扭曲
- 过度清洗：无任何有价值信息丢失
- 可读性与结构：表达通顺、格式规范、结构完整
- 事实准确性：所有保留的事实完全正确

**示例**：
1. 仅去除多余空格和重复标点
2. 统一日期格式（如"2024年03月15日" → "2024-03-15"）
3. 完全无改动

---

### 0.75 —— 轻微问题

**定义**：存在轻微瑕疵，但不影响核心理解和决策。

**判断标准**：
- 语义一致性：核心含义保留，仅边缘信息有微小偏差
- 过度清洗：删除了辅助性细节，但核心信息完整
- 可读性与结构：表达基本通顺，有极轻微的格式问题
- 事实准确性：核心事实正确，仅非关键数据有微小偏差

**示例**：
1. 删除了括号内的补充说明（如"（含技术路线说明）"），但主要信息保留
2. 英文术语翻译为中文（如"API Gateway" → "API网关"），含义不变
3. 段落顺序微调（如背景和目标互换），逻辑仍通顺

---

### 0.50 —— 中等问题

**定义**：存在明显问题，影响部分理解和决策。

**判断标准**：
- 语义一致性：部分含义丢失或扭曲，但主旨仍在
- 过度清洗：删除了有价值的信息，但核心框架保留
- 可读性与结构：表达有明显问题，或结构有部分破坏
- 事实准确性：有事实错误，但非核心数据

**示例**：
1. 删除了各要点的具体措施和量化目标，但核心方向保留
2. 列表转为段落，材料名称保留但层次丢失
3. 机构名从法定全称缩写为简称
4. 法律条款中"书面同意"简化为"同意"

---

### 0.25 —— 严重问题

**定义**：存在严重问题，严重影响理解和决策。

**判断标准**：
- 语义一致性：核心含义严重丢失或扭曲
- 过度清洗：删除了关键信息，导致理解困难
- 可读性与结构：表达严重不通顺，或结构严重破坏
- 事实准确性：有严重事实错误（核心数据被篡改）

**示例**：
1. 三个具体要点压缩为一句话
2. 技术参数从±1100误改为±1000
3. 表格被完全破坏，转为纯文本
4. 过度缩写导致难以理解（如"登录信息" → "登录信"）

---

### 0.00 —— 完全不合格

**定义**：该维度存在致命问题，完全无法接受。

**判断标准**：
- 语义一致性：含义完全丧失或严重扭曲
- 过度清洗：几乎所有有价值信息被删除
- 可读性与结构：完全不可读或结构完全丧失
- 事实准确性：核心事实完全错误

**示例**：
1. 原文三个要点全部删除，仅剩"本次进行了升级"
2. 营业收入从125.8亿改为128.5亿（核心数据完全篡改）
3. 法律条款的限定条件、例外条款、兜底条款全部删除

## 第四章：扣分规则

### 4.1 各错误类型的扣分幅度

| 错误类型 | 严重程度 | 建议评分范围 |
|---|---|---|
| 删除完整句子（含关键信息） | critical | 0.00-0.25 |
| 删除完整句子（仅辅助信息） | major | 0.25-0.50 |
| 删除从句（含关键信息） | major | 0.25-0.50 |
| 删除从句（仅修饰信息） | minor | 0.50-0.75 |
| 删除短语（含关键信息） | major | 0.25-0.50 |
| 删除短语（仅修饰信息） | minor | 0.50-0.75 |
| 修改数值（核心数据） | critical | 0.00-0.25 |
| 修改数值（非核心数据） | major | 0.25-0.50 |
| 修改日期 | critical | 0.00-0.25 |
| 修改名称（法定全称） | major | 0.25-0.50 |
| 修改技术参数 | critical | 0.00-0.25 |
| 修改引用/条文 | critical | 0.00-0.25 |
| 表格转纯文本 | critical | 0.00-0.25 |
| 列表转段落 | major | 0.25-0.50 |
| 段落顺序调整 | minor | 0.50-0.75 |
| 过度缩写（严重影响理解） | critical | 0.00-0.25 |
| 过度缩写（轻微影响理解） | major | 0.25-0.50 |
| 不合理脱敏 | major | 0.25-0.50 |

### 4.2 同维度多错误处理

**规则**：首先根据最严重错误确定基础等级；若同一维度存在多个同等级错误，可在该等级内向下调整，但不得跨越下一个离散等级。

**示例**：
- 1 个 major → 0.50
- 2 个 major → 0.50（等级内不降）
- 3 个及以上 major → 0.25（降一个等级）
- 1 个 critical + 多个 major → 0.00-0.25（以 critical 为准）

**具体阈值**：
| 同等级错误数量 | 调整规则 |
|---|---|
| 1 个 | 维持该等级对应分数 |
| 2 个 | 维持该等级对应分数 |
| ≥ 3 个 | 降一个等级（如 3 个 major → 0.25） |
| 1 个更严重 + N 个次严重 | 以更严重等级为准 |

### 4.3 合理治理 vs 过度清洗的边界

以下情况属于**合理治理**，过度清洗维度应评 1.00：
- 删除多余空格、重复标点
- 统一标点符号（中英文标点混用 → 统一）
- 统一日期格式（多种格式 → ISO 格式）
- 合理脱敏（隐去身份证号、手机号等个人信息）
- OCR 修复（修正识别错误）
- 删除明显冗余（同一句话重复出现）

以下情况属于**过度清洗**，应按严重程度扣分：
- 删除具体措施、量化目标
- 删除限定条件、前提假设
- 删除豁免条款、例外情况
- 删除括号内的实质性补充说明
- 不合理脱敏（隐去业务信息、技术参数）

## 第五章：优先级与冲突解决

### 5.1 维度独立评分

每个维度独立打分，不因其他维度的分数而调整。同一错误可能影响多个维度（见第二章归属表），但每个维度的扣分幅度独立计算。

### 5.2 多维度冲突

当不同维度的评分产生矛盾时，按以下优先级处理：

**优先级顺序**（从高到低）：
1. **事实准确性**：任何事实错误都是硬伤，不能因其他维度良好而忽略
2. **语义一致性**：核心含义丢失是严重问题
3. **过度清洗**：信息丢失影响内容完整性
4. **可读性与结构**：格式问题相对最轻

**判定阈值**（可配置）：

| 判定 | 默认条件 | 说明 |
|---|---|---|
| pass | 四个维度均 ≥ 0.75 且加权总分 ≥ 0.80 | 阈值可在实验中调整 |
| review | 不满足 pass 和 fail 条件 | 中间地带 |
| fail | 任一维度 ≤ 0.25 或加权总分 < 0.50 | 阈值可在实验中调整 |

**加权总分公式**：

加权总分 = 语义一致性 × 0.30 + 过度清洗 × 0.30 + 可读性与结构 × 0.20 + 事实准确性 × 0.20

**注意**：pass/fail 的阈值（0.80、0.50、0.25）为默认值，论文实验中可通过配置文件调整，以便进行阈值敏感性分析。

## 第六章：Judge Decision Protocol（判定决策协议）

本协议定义 LLM Judge 和人工标注员的标准化评分流程。按照以下 5 个问题顺序执行，每个问题的答案决定下一步操作。

### Question 1：有没有事实错误？

**检查项**：
- 逐项核对治理后文本中仍然保留的数值、日期、名称、参数
- 与治理前文本对比，是否有任何不一致

**决策**：
- 发现事实错误 → 事实准确性评 0.00-0.25，进入 Question 2
- 无事实错误 → 事实准确性评 1.00，进入 Question 2

### Question 2：语义有没有变化？

**检查项**：
- 治理后文本的信息意图是否改变
- 命题意义是否改变
- 因果关系、条件关系是否改变

**决策**：
- 语义改变 → 语义一致性评 0.25-0.50，进入 Question 3
- 语义未变 → 语义一致性评 0.75-1.00，进入 Question 3

### Question 3：有没有删除有价值的信息？

**检查项**：
- 列出所有被删除的内容
- 判断每项删除是否属于"有价值信息"
- 排除合理治理（空格、标点、格式）

**决策**：
- 删除了关键信息 → 过度清洗评 0.00-0.25，进入 Question 4
- 删除了部分信息 → 过度清洗评 0.25-0.50，进入 Question 4
- 仅删除冗余 → 过度清洗评 0.75-1.00，进入 Question 4

### Question 4：有没有结构或格式问题？

**检查项**：
- 表格、列表是否完整
- 标题层级是否合理
- 表达是否通顺
- 排版是否规范

**决策**：
- 结构严重破坏 → 可读性与结构评 0.00-0.25，进入 Question 5
- 结构部分破坏 → 可读性与结构评 0.25-0.50，进入 Question 5
- 轻微格式问题 → 可读性与结构评 0.50-0.75，进入 Question 5
- 无问题 → 可读性与结构评 1.00，进入 Question 5

### Question 5：输出最终结果

**操作**：
1. 汇总四个维度的分数
2. 检查是否存在同维度多错误，按第四章规则调整
3. 计算加权总分
4. 按阈值确定判定结果（pass/review/fail）
5. 记录评分理由（rationale）
6. 记录瑕疵清单（如有）

## 第七章：人工标注流程

### Step 1：通读原文

1. 阅读治理前文本，理解其**主题、结构、关键信息**
2. 标记关键事实：数字、日期、机构名、法律引用
3. 标记结构元素：表格、列表、标题、段落

### Step 2：对比差异

1. 逐句对比治理前和治理后文本
2. 列出所有差异点
3. 对每个差异点分类：
   - 合理治理（不扣分）
   - 轻微问题
   - 中等问题
   - 严重问题

### Step 3：按 Decision Protocol 评分

严格按照第六章的 Question 1-5 顺序执行。

### Step 4：确定最终评分

1. 检查四个维度之间是否有矛盾
2. 按优先级规则解决冲突
3. 检查同维度是否有多个错误，按规则调整
4. 计算加权总分
5. 确定判定结果

### Step 5：标注瑕疵清单

对每个扣分项，记录：
- 类别（按 Error Taxonomy 分类）
- 严重程度（critical / major / minor）
- 描述
- 位置（段落 ID、字符偏移、原文片段）

## 第八章：一致性校验

### 8.1 标注员间一致性

- 任意两个标注员对相同样本的评分，Spearman 秩相关系数应 ≥ 0.8
- 加权 Kappa 系数应 ≥ 0.6
- 组内相关系数（ICC）应 ≥ 0.7
- 判定结果（pass/review/fail）的一致率应 ≥ 0.8
- 如不一致，由第三位标注员仲裁

### 8.2 标注员内一致性

- 同一标注员在不同时间对相同样本的评分，方差应 < 0.05
- 建议间隔至少一周后重新标注部分样本

### 8.3 与 LLM 一致性

- LLM 评分与人工评分的 Spearman 秩相关系数应 ≥ 0.8
- 加权 Kappa 系数应 ≥ 0.6
- LLM 判定结果与人工判定的一致率应 ≥ 0.7

**说明**：使用 Spearman 而非 Pearson 的原因：评分是离散的有序变量（1.00/0.75/0.50/0.25/0.00），Spearman 秩相关更适合有序数据的一致性分析。Pearson 相关系数作为辅助指标报告。
