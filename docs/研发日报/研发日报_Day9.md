### 2.3 组长今日工作内容

1. **全指标复核**：逐一验证 9 项指标（Pearson/Spearman/Kappa/Kendall's W/ROUGE-L/BERTScore/F1/锚点/偏置）的计算逻辑和最新数值，确认 I1-I8 全部达标。
2. **性能优化**：分析 EvalCallbackHandler 延迟数据，精简 Prompt（减少约 200 tokens，-17%），调整 max_tokens（2048->1024），响应时间从 3.2s 降至 2.7s（-16%）。
3. **约束词删除检测**：在 Prompt 中新增约束词检测规则——当治理后文本删除了"不得""必须""应当""不少于""不超过""至少""最高""最低""以内""以上"等约束词时，即使"大意保留"，factual <= 0.4、semantic <= 0.5。
4. **定义条款变更检测**：在 Prompt 中新增定义条款检测规则——当治理后文本删除或泛化了"书面""口头""签署前后""明确标注""保密""机密"等定义性限定词时，属于定义条款弱化，factual <= 0.4、semantic <= 0.5。
5. **Profile 实体严格性**：为政务通告模式和法律文书模式增加了实体名称严格性规则——政务模式要求机构名必须使用法定全称、技术参数必须精确；法律模式要求定义条款逐字保留、限定词不得删除。
6. **校准脚本修复**：修复 `run_calibration.py` 中 `human_score` 字段名不匹配的问题——数据集使用 `human_scores`（含四维度分解），脚本改为通过加权公式（semantic x 0.35 + factual x 0.35 + structure x 0.15 + readability x 0.15）计算总分。
7. **验收标准自动判定验证**：确认 `reporter.py` 中 `_generate_checklist()` 的 I1-I8 判定逻辑与研发计划书一致。

**计划工时：6h　　实际工时：约 6h　　偏差：持平**
