## 三、今日研发过程记录

### 3.1 高偏差样本分析

今天首先完成了对首轮校准高偏差样本的集中复盘。分析结果显示，当前模型误判主要集中在以下三类场景：

| 偏差类型 | 具体表现 | 初步原因 |
| --- | --- | --- |
| 数值敏感性不足 | 对金额、比例、数量级误改扣分不够 | Prompt 对关键数值变化的严重性强调不强 |
| 格式优化误罚 | 把标点、空格、换行调整误认为语义变动 | 缺少明确的格式豁免说明 |
| 过度清洗识别不足 | 对大段删除、信息压缩过度的问题处罚偏轻 | 对结构性删减的严重程度约束偏弱 |

通过这一步，把原本比较模糊的"模型评分不够贴近人工判断"进一步拆解成几个具体问题，为后续调优提供明确方向。

### 3.2 Prompt 调优的主要方向

在问题分析完成后，今天重点从以下几个方面对评分规则进行了优化：

#### （1）强化关键数值类问题的判定

对金额、比例、年份、数量级等涉及事实准确性的内容进一步提高敏感度，要求模型优先检查这类信息是否发生变化，并在出现明显误改时给出更严格的处罚判断。

#### （2）增加格式性调整的豁免规则

针对纯标点规范、空格调整、换行优化、段落对齐等治理操作，明确其属于形式层面的整理，不应被误判为语义变化，从而减少不必要的扣分。

#### （3）提高过度清洗场景的严重程度

对于整段内容被删除、压缩后丢失关键信息、文本结构被明显削弱等情况，在规则中补强其严重等级，使模型能够更准确识别"内容保真度受损"的场景。

#### （4）细化"合理改写"与"错误改写"的边界

通过补充边界定义，让模型更清楚哪些变化属于表达优化，哪些变化已经实质影响原文含义，从而提升相似样本之间的判断一致性。

### 3.3 多轮验证过程

在完成上述调优之后，继续开展了多轮校准验证，以确认优化不是偶然生效，而是能够在整体样本上持续发挥作用。整个验证过程大致分为两个阶段：

- **第一阶段**：完成第一轮调优后重新校准，观察结果是否比 Day 1 首轮测试更接近人工评分；
- **第二阶段**：针对仍存在波动的边界样本继续补充规则，再次开展验证，进一步提升规则稳定性。

这一过程说明，今日工作并不是"一次修改、一次测试"的简单线性过程，而是一个不断发现问题、修正规则、再用数据验证的循环过程，也更加符合实际研发中 Prompt 调优的典型特点。
