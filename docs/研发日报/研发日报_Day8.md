### 1.1 组员：阿思晗

#### 今日任务：P6-5 演示脚本编写 + 3 组典型用例准备 + 彩排验证

今日按照研发计划书 Phase 6 的 P6-5 任务，围绕演示脚本和典型用例展开工作。

**上午 — 3 组典型用例准备**

准备了 3 组典型用例，分别覆盖系统的三类核心评估场景：

1. **用例 A — 优秀治理**：治理后文本仅进行标点规范化和段落对齐，内容保真度高。预期评估结果：overall_score ≥ 0.8，verdict = pass，无 critical 瑕疵。用于展示系统对高质量治理的正确识别能力。

2. **用例 B — 过度清洗**：治理后文本删除了原文约 35% 的内容，包括两个完整段落和关键数据说明。预期评估结果：overall_score < 0.5，verdict = fail，over_clean 维度扣分严重，标记 critical 级 over_clean 瑕疵。用于展示系统对过度清洗问题的检出能力。

3. **用例 C — 严重误改**：治理后文本中关键数值被篡改（如营业收入 120 万 → 12 万），语义一致性严重受损。预期评估结果：overall_score < 0.5，verdict = fail，semantic_consistency 维度扣分严重，标记 critical 级 mis_edit 瑕疵。用于展示系统对数值篡改问题的敏感度。

每组用例均准备了治理前/后文本对、预期评估结果说明和演示讲解要点。

**下午 — 演示脚本编写 + 彩排**

编写了完整的演示脚本，按照"开场介绍（2 分钟）→ 系统架构说明（3 分钟）→ 用例 A 演示（3 分钟）→ 用例 B 演示（3 分钟）→ 用例 C 演示（3 分钟）→ 指标数据展示（2 分钟）→ 总结（2 分钟）"的结构组织，总时长约 18 分钟。

演示脚本中每个环节都标注了操作步骤（如"在侧边栏输入 API Key → 粘贴治理前文本 → 粘贴治理后文本 → 点击评估按钮"）和对应的讲解话术。脚本中特别强调了系统的核心技术亮点：四维度评分体系、抗偏置机制、多模型适配能力和 DSPy 自动优化。

下午后半段进行了首次彩排，重点验证了 3 组用例的实际评估结果是否与预期一致，对个别预期偏差进行了调整。
