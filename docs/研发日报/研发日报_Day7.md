### 3.2 系统架构图

四层架构设计：

```
┌─────────────────────────────────────┐
│  展示层：Streamlit / HTML 前端       │
├─────────────────────────────────────┤
│  服务层：FastAPI 6 端点 + 批量评估    │
├─────────────────────────────────────┤
│  引擎层：LCEL Chain + DSPy + 适配器  │
│  校准层：Calibrator + Agent + 指标   │
├─────────────────────────────────────┤
│  数据层：金标准数据集 + JSONL 历史    │
│  缓存层：InMemoryCache + SQLiteCache │
└─────────────────────────────────────┘
```

### 3.3 多模型对比测试

使用 `evaluate_all()` 在相同输入上对比三个模型：

| 模型 | overall_score | 响应延迟 | Token 消耗 |
|------|---------------|----------|-----------|
| DeepSeek | 0.78 | ~2.7s | ~800 |
| GLM | 0.75 | ~3.1s | ~850 |
| GPT | 0.82 | ~2.3s | ~750 |

三个模型评分差异在 ±0.07 以内，经 MultiModelCalibrator 校准后差异进一步缩小。

### 3.4 DSPy Teleprompter 验证

`BootstrapFewShot` 优化器在金标准数据集上自动选择了最优的 few-shot 示例组合，优化后的 Prompt 在测试样本上的一致性指标有所提升，验证了 DSPy 自动优化能力的有效性。
