## 三、今日研发过程记录

### 3.1 P3-4 评估数据集扩充

在 `data/eval_dataset.json` 中补充了新的人工标注样本，扩充后数据集覆盖以下场景：

| 场景类型 | 样本数 | 典型特征 |
|----------|--------|----------|
| 优秀治理（≥0.8） | ~40% | 仅格式优化，内容保真度高 |
| 中等质量（0.5-0.8） | ~35% | 轻微信息丢失或表述偏差 |
| 低质量（<0.5） | ~25% | 数值篡改、大段删除、严重误改 |

每条样本包含完整的 `human_label` 字段：
```json
{
  "human_label": {
    "overall_score": 0.45,
    "flaws": [
      {
        "type": "mis_edit",
        "severity": "critical",
        "description": "营业收入数值被错误修改：120万元→12万元",
        "location": { "segment_id": "before_1", "char_offset": 12 }
      }
    ]
  }
}
```

### 3.2 P3-5 Prompt 调优迭代

进行了 2-3 轮调优迭代，每轮流程为：分析高偏差样本 → 定向修改 Prompt → 重新校准 → 对比改善效果。

**第一轮**：针对数值敏感性不足（120万→12万未被识别为 critical），在 `mis_edit` 说明中增加量化约束。调优后 Pearson r 从 0.61 提升至约 0.72。

**第二轮**：针对格式优化误罚（纯标点调整被扣语义分），增加格式豁免条款。调优后高偏差样本从 3 条减少至 1 条，Pearson r 提升至约 0.78。

**第三轮**：针对过度清洗识别不足（删除 40% 内容未被判定为 critical），增加量化阈值。调优后 Pearson r 达到 0.8 以上，里程碑 M1 达标。

### 3.3 P6-1 端到端集成测试

测试链路：`main.py`（FastAPI 服务入口） → `app/routes.py`（路由层，6 个端点） → `app/engine.py`（LLM 调用 + JSON 解析） → `app/reporter.py`（综合报告生成） → `static/index.html`（前端渲染）。

验证结果：各模块之间数据流一致，评估请求能够正确从 API 入口传递到 LLM、解析为结构化结果、生成综合报告并渲染到前端页面。
