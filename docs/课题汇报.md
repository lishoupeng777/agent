## 一、课题背景与研究动机

### 1.1 问题是什么

自动化数据治理（去噪、修复、归一化）会对原始文本进行改写，但"改写"本身是有风险的：

- **过度清洗**：删掉了有效信息（如删除完整段落、压缩关键内容）
- **误改语义**：修改了数值、日期、人名等关键事实（如 120万→12万）
- **结构破坏**：打乱了原始文本的排版、表格、层级关系

要让治理结果可信，必须有一种能力来评估"治理前后语义是否一致、是否过度改写、可读性与结构是否合格"。

### 1.2 为什么用 LLM 做裁判

传统规则方法（如关键词匹配、编辑距离）难以理解语义层面的变化。用大模型做裁判（LLM-as-Judge）是当前评估范式的前沿方向，但其公认的难点在于：

- **评分稳定性**：同一输入多次评估，分数是否一致？
- **人类一致性**：模型评分与人工判断是否吻合？
- **可解释性**：能否给出让人信服的判定理由？
- **抗偏置**：是否会因文本长度、瑕疵位置等因素产生系统性偏差？

### 1.3 本课题的差异化定位

| 对比维度 | 开源通用评测 | 本课题 |
|----------|-------------|--------|
| 场景 | 一般生成质量打分 | 治理前后比对（保真度） |
| 研究对象 | 打分本身 | 裁判本身的可信度 |
| 瑕疵定位 | 无 | 锚点级定位（回溯到具体位置） |
| 可复现性 | 一般不要求 | SHA256 令牌，固定策略可复现 |
| 抗偏置 | 一般不涉及 | 长度/位置偏置检测与缓解 |

**一句话概括：** 本课题不是做"打分工具"，而是做"可信赖的裁判"——面向治理前后比对这一特定场景，把裁判的"稳、准、可解释、可复现"作为研究核心。

## 二、研究目标

设计并实现一个内容保真度与治理质量评估智能体：

> 抽样比对治理前后文本，判定语义一致性、识别过度清洗与误改、评估可读性与结构质量，输出可锚点定位、可复现的评估结果（维度评分 + 锚点级瑕疵清单 + 判定理由）。作为服务被质检环节按需调用。

## 四、核心能力实现

### 4.1 语义一致性评估

**目标：** 治理前后比对，判定是否保真。

**实现方式：**

- **Prompt 维度定义**：在 System Prompt 中定义"语义一致性"维度（权重 0.35），要求 LLM 检查核心事实、实体名称、量化指标是否被误改或丢失
- **规则引擎补充**：`detect_flaws()` 通过字符级 diff 检测数值变化（变化幅度 > 10% 自动标记为 critical 级 `mis_edit`）
- **Profile 增强**：政府公告 Profile 对日期/期限/义务敏感，法律文书 Profile 对条款引用/权利义务敏感

**评分标准：**

| 分数区间 | 含义 |
|----------|------|
| 0.9 – 1.0 | 完全保真，仅做格式优化 |
| 0.7 – 0.9 | 轻微表述变化，核心事实无损 |
| 0.5 – 0.7 | 存在信息丢失或轻微误改 |
| 0.3 – 0.5 | 关键事实被篡改或大量内容被删 |
| 0.0 – 0.3 | 语义严重偏离，治理基本失败 |

### 4.2 过度清洗/误改识别

**目标：** 定位被错误删除或改写的内容。

**实现方式：**

- **过度清洗（over_clean）检测**：
  - 规则引擎：段落级匹配率 < 0.4 判定为删除，删除超 30% 字符数标记为 critical
  - LLM 判断：评估被删除内容是否包含有效信息

- **误改（mis_edit）检测**：
  - 规则引擎：字符级 diff 提取数值变化，变化幅度 > 10% 标记为 critical
  - LLM 判断：评估修改是否改变了原文含义

- **瑕疵结构化输出**：

```json
{
  "category": "mis_edit",
  "severity": "critical",
  "description": "营业收入数值被错误修改：120万元→12万元，丢失一个数量级",
  "location": {
    "segment_id": "before_1",
    "start_char": 12,
    "end_char": 24,
    "snippet": "营业收入为120万元"
  },
  "suggestion": "恢复原始数值或标注修改原因"
}
```

### 4.3 可读性与结构质量评估

**目标：** 标题树合理性、段落连贯性、表格可读性。

**实现方式：**

- **Prompt 维度定义**："结构完整性"（权重 0.15）+ "可读性"（权重 0.15）
- **规则引擎**：检测 Markdown 表格行/列丢失、列表层级破坏、标题层级错乱
- **格式豁免**：纯标点/空格/段落对齐调整不扣语义分（防止格式优化被误罚）

### 4.4 锚点级瑕疵定位

**目标：** 评估结果均可回溯到具体位置。

**实现方式：**

- **对齐模块**：`build_anchored_text()` 将前后文本按段落对齐，用 `difflib.SequenceMatcher`（阈值 0.4）匹配段落，加上 `[Before N]`/`[After N]` 锚点标记
- **定位精度**：每个瑕疵输出 `segment_id` + `start_char` + `end_char` + `snippet`
- **准确率计算**：`compute_anchor_accuracy()`，snippet 重叠 ≥ 4 字符 + start_char 容差 ≤ 10 字符

### 4.5 评分稳定性与可解释性

**目标：** 降低评分波动、给出判定理由。

**实现方式：**

- **稳定性验证**（`app/stability.py`）：
  - 同一输入跑 N 次（默认 5 次，temperature=0.0）
  - 计算每维度均值、方差、标准差
  - 方差 < 0.005 判定为稳定
  - 瑕疵投票机制：按 (类别, 片段前30字符) 分组，多数投票决定严重程度

- **可解释性**：
  - 每个维度输出 `reason` 字段（判定理由）
  - 每个瑕疵输出 `description`（问题描述）+ `suggestion`（修改建议）
  - 置信度计算：三因子加权（diff 一致性 + reason 完整性 + 规则一致性）

- **临界区自动重采样**：评分落在 0.75-0.85 时自动多跑 2 次取平均，减少边界误判

### 4.6 可复现性

**目标：** 固定策略下，相同输入始终得到相同输出。

**实现方式：**

- SHA256 哈希生成可复现令牌：`SHA256(前后文本 + 温度 + 模型 + Prompt版本 + 规则版本 + Profile)`
- 相同输入在固定策略下始终生成相同令牌
- 支持按令牌查找历史评估结果（`storage.py`）

## 五、研究亮点：评分稳定化 + 一致性校准

这是本课题的核心研究亮点，给出了可验证的量化证据。

### 5.1 一致性校准

**方法：** 在 34 条人工标注数据集上，对比 LLM 评分与人工评分，计算 5 项指标。

**校准指标：**

| 指标 | 说明 | 目标值 |
|------|------|--------|
| Pearson r | 线性相关系数 | ≥ 0.8 |
| Spearman ρ | 秩相关系数 | ≥ 0.8 |
| MAE | 平均绝对误差 | 尽量低 |
| RMSE | 均方根误差 | 尽量低 |
| 一致率 | 误差 ≤ 0.1 的比例 | ≥ 80% |

**校准过程（数据驱动的迭代优化）：**

| 轮次 | 操作 | Pearson r |
|------|------|-----------|
| 基线 | 首次校准 | 0.61 |
| 第一轮 | 强化数值敏感性 | 0.72 |
| 第二轮 | 增加格式豁免条款 | 0.78 |
| 第三轮 | 提高过度清洗权重 | **≥ 0.8** |

**偏差归因方法：** 对高偏差样本逐条分析，将误判归纳为三类（数值敏感性不足 / 格式优化误罚 / 过度清洗识别不足），每轮定向修改 Prompt 中对应的规则。

### 5.2 评分稳定化

**方法：** 同一输入执行 N 次评估（temperature=0.0），计算方差。

**稳定化机制：**

- 低温度（temperature=0.0）减少随机性
- 多次采样取平均（stabilize 模式）
- 瑕疵投票：多数投票决定严重程度，避免单次波动
- 临界区重采样：边界分数自动多跑 2 次

### 5.3 抗偏置

**偏置检测：**

| 偏置类型 | 检测方法 | 风险阈值 |
|----------|----------|----------|
| 长度偏置 | 前后文本长度比 | <0.3 高风险，<0.6 中风险 |
| 位置偏置 | 瑕疵前后半部分分布 | >80% 集中 = 偏置 |

**偏置缓解：**

- 抗偏置 Prompt 指令自动注入（长度无关/位置无关/格式无关/领域无关）
- 偏置缓解得分 = 维度分数标准差 × 0.5 + 长度因子 × 0.5

## 九、总结

本课题围绕"LLM-as-Judge 裁判可信度"这一核心问题，构建了一个面向治理前后比对场景的评估智能体。系统的核心贡献在于：

1. **双通道瑕疵检测**：规则引擎（确定性）+ LLM 语义判断（灵活性）互为补充，数值变化等关键瑕疵不遗漏
2. **数据驱动的 Prompt 调优**：通过"偏差归因→定向修改→校准验证"的迭代循环，Pearson r 从 0.61 提升至 ≥ 0.8
3. **评分稳定化机制**：多次采样 + 瑕疵投票 + 临界区重采样，方差 < 0.005
4. **锚点级可追溯**：每个瑕疵精确定位到 segment + 字符位置，支持人工复核
5. **可复现性保障**：SHA256 令牌确保固定策略下结果一致
6. **可作为服务调用**：标准 REST API，支持批量评估、多模型切换、差异化 Profile

系统已完成核心功能开发与验收测试，具备被质检流水线集成调用的能力。
